# WebLinkService (HTA Alternative for Modern Windows)

Легкий HTTP-сервер на **Delphi 7 Lite**, который выступает мостом между браузером и файловой системой ОС. Создан как современная замена устаревшей технологии HTA.

### Зачем это нужно?
Современные браузеры из соображений безопасности не дают JS-скриптам работать с файлами на диске напрямую. Эта утилита снимает ограничения, позволяя веб-интерфейсу управлять локальными данными через простой HTTP-протокол.

### Основной функционал (Файловый мост)
* **Чтение файла (GET):** При запросе `GET /myfile.txt` сервис отдает содержимое реального файла `myfile.txt` из рабочей директории.
* **Запись файла (POST):** При запросе `POST /myfile.txt` данные из `payload` (тело запроса) напрямую перезаписывают файл на диске.

### Как это работает (Quick Start)
1. Положите ваш `index.html` (или любой другой файл) в папку с `WebLink.exe`.
2. Запустите `WebLink.exe`.
3. Откройте в браузере: `http://127.0.0.1:50086/index.html`.

Любой запрос к корню (например, `/main.css`) подтянет файл из папки, где лежит исполняемый файл.

**Пример работы с данными:**
* Через **GET** читаем `/myappdata.json`.
* Через **POST** отправляем новые данные в теле запроса — файл физически перезаписывается (если нет флага *ReadOnly*).

### Технические детали
* **Движок:** Delphi 7 + WinAPI + Indy 9.0.18.
* **Совместимость:** Windows XP, 7, 10 — проверено, работает стабильно. (Win11 — теоретически да).
* **Конфиги:** Важно сохранять `CRLF` в файлах настроек (`.cfg`), Git настроен на сохранение окончаний строк.

### Для кого это
Для разработчиков локальных инструментов, которым нужно простое управление файлами через веб-интерфейс без развертывания тяжелых стеков типа Node.js или Python.
